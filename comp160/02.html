This session introduces core UML diagrams that you will use to describe the architecture of the software that you have designed. Documenting source code and leveraging it to both, manage complexity, and, to avoid re-inventing solutions to well-known problems, will serve you well in the future.


Studio Practice

Each week you will engage in mandatory studio practice under the supervision of mentors who will help you to develop your collaborative programming practice. You are also expected to attend a mandatory meeting with your product owner each week.


Workshop
Learning Objectives:
  • Illustrate a class design using a UML diagram
  • Assess the efficiency of object interaction using a UML diagram
  • Apply knowledge of UML to describe a game architecture
