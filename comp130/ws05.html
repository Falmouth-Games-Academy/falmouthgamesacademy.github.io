This session introduces the basics of relational databases and how they apply to games. You will follow tutorial to create a high score table for a game. Data is ubiquitous in games! From complex assets, levels, and narrative systems to data that needs to be collected from players in both single-player games (e.g. RPGs) and multiplayer games (e.g. levels, scores, etc.), data is everywhere. As such, the importance of data management and databases should be clear.


Studio Practice

Each week you will engage in mandatory studio practice under the supervision of mentors who will help you to develop your collaborative programming practice. You are also expected to attend a mandatory meeting with your product owner each week.


Workshop
Learning Objectives:
  • Explain why a database might be used in a game architecture
  • Illustrate a normalised database design using an entity-relationship diagram
  • Design and Implement a database using SQL
  • Write a program that queries a database using SQL
