Exceptions are a useful programming structure for diagnosing error conditions during the development of a program, and for writing programs that can recover from such error conditions at runtime. When diagnosing a problem, the debugger is an essential software tool which takes the guesswork out of determining the cause of a bug.

Learning Objectives:
  • Explain the usefulness of exceptions and assertions in writing robust software
  • Write Python programs that throw and catch exceptions
  • Use the PyCharm debugger to trace the execution of programs
